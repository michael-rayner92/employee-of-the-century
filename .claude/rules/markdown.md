# 📝 Markdown style rules

## 😀 Headers use emojis

Every markdown header (`#` through `######`) must start with an emoji relevant to the section's content, followed by a space, then the header text — matching the existing style in [README.md](../../README.md) (e.g. `## ✨ Features`, `## 🚀 Running locally`, `## 🗂️ Project structure`).

- Pick an emoji that reflects the section's meaning (🚀 for running/deploying, 🧩 for architecture, 🎛️ for controls/config, 🛠️ for tooling), not a random decoration.
- Apply this when creating new markdown files and when adding new headers to existing ones.
